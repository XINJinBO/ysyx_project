# MiniRV RISC-V 处理器八条指令设计分析

## 1. 概述

MiniRV 是一个基于 Logisim-evolution 构建的最小化 RISC-V 处理器，实现了 RV32I 指令集中的八条基本指令。该设计采用单周期数据通路架构，每条指令在一个时钟周期内完成执行。

### 1.1 支持的指令集

| 指令   | 类型    | 功能描述           | 操作码 | funct3 |
| ------ | ------- | ------------------ | ------ | ------ |
| ADDI   | I-Type  | 立即数加法         | 0x13   | 000    |
| JALR   | I-Type  | 跳转并链接寄存器   | 0x67   | 000    |
| ADD    | R-Type  | 寄存器加法         | 0x33   | 000    |
| LUI    | U-Type  | 加载高位立即数     | 0x37   | -      |
| LW     | I-Type  | 加载字             | 0x03   | 010    |
| LBU    | I-Type  | 加载无符号字节     | 0x03   | 100    |
| SW     | S-Type  | 存储字             | 0x23   | 010    |
| SB     | S-Type  | 存储字节           | 0x23   | 000    |

---

## 2. 处理器架构

### 2.1 寄存器文件

- 16 个 32 位通用寄存器（x0-x15）
- 4 位寄存器地址（可寻址 16 个寄存器）
- 双端口读、单端口写设计
- x0 硬编码为零

### 2.2 主要功能单元

| 组件             | 规格                         | 功能                               |
| ---------------- | ---------------------------- | ---------------------------------- |
| 程序计数器 (PC)  | 32 位 Counter                | 提供指令地址，支持复位和跳转       |
| 指令存储器 (ROM) | 24 位地址，32 位数据         | 存储程序指令                       |
| 数据存储器 (RAM) | 24 位地址，32 位数据         | 存储程序数据，支持字节使能         |
| ALU              | 加法器、移位器、比较器       | 执行算术和逻辑运算                 |

## 3. 指令格式

### 3.1 I-Type（立即数型）

```
 31       20 19   15 14  12 11    7 6     0
+-----------+-------+------+-------+-------+
| imm[11:0] |  rs1  |funct3|  rd   |opcode |
+-----------+-------+------+-------+-------+
     12位      5位     3位    5位     7位
```

用于：ADDI, JALR, LW, LBU

### 3.2 R-Type（寄存器型）

```
 31    25 24  20 19  15 14  12 11    7 6     0
+-------+-------+-------+------+-------+-------+
|funct7 |  rs2  |  rs1  |funct3|  rd   |opcode |
+-------+-------+-------+------+-------+-------+
   7位     5位     5位     3位    5位     7位
```

用于：ADD

### 3.3 U-Type（立即数长型）

```
 31           12 11    7 6     0
+---------------+-------+-------+
| imm[31:12]    |  rd   |opcode |
+---------------+-------+-------+
      20位        5位     7位
```

用于：LUI

### 3.4 S-Type（存储型）

```
 31       25 24  20 19  15 14  12 11     7 6     0
+-----------+-------+-------+------+--------+-------+
| imm[11:5] |  rs2  |  rs1  |funct3|imm[4:0]|opcode |
+-----------+-------+-------+------+--------+-------+
    7位       5位     5位     3位     5位      7位
```

用于：SW, SB

## 4. 八条指令详细分析

---

### 4.1 JALR（跳转并链接寄存器）⭐ 重点

**功能描述**：

```
temp      = (rs1 + sign_extend(imm[11:0])) & ~1
rd        = PC + 1
PC        = temp
```

**指令编码**：`I-Type | imm[11:0] | rs1[4:0] | funct3=000 | rd[4:0] | opcode=1100111(0x67)`

#### 4.1.1 设计思路详解

**1. 为什么需要 JALR？**

JALR 是 RISC-V 中实现**间接跳转**的核心指令，主要用于：

- **函数返回**：从栈中弹出返回地址并跳转
- **函数指针调用**：通过寄存器中的地址进行调用
- **switch-case 实现**：跳转表（Jump Table）

与 JAL 的区别：

- JAL 使用 PC 相对寻址（地址 = PC + offset）
- JALR 使用寄存器间接寻址（地址 = rs1 + offset）

**2. 立即数符号扩展**

```
12 位立即数 ──→ Bit Extender(12→32) ──→ 32 位有符号数
```

- 12 位立即数范围：-2048 到 2047
- 符号扩展确保负偏移量的正确处理

**3. 目标地址计算**

```
  rs1_data ──┐
             ├──→ 32 位 Adder ──→ 目标地址（未对齐）
  imm_ext  ──┘
```

- 加法器将 rs1 的值与扩展后的立即数相加

**4. 地址对齐处理**

```
目标地址 ──→ AND Gate(& ~1) ──→ 对齐后的地址（最低位清零）
```

- RISC-V 要求跳转目标地址必须是 2 字节对齐
- 通过与 `~1` 进行 AND 操作清除最低位（bit[0] = 0）
- 确保跳转目标地址始终为偶数地址

**5. 返回地址保存**

```
PC_data ──→ Adder(+1) ──→ PC + 1（返回地址）
                              │
                              ▼
                    Controlled Buffer ──→ write_data ──→ 寄存器堆[rd]
```

- 将当前 PC + 1 保存到目标寄存器 rd
- 通常 rd = x1（ra，返回地址寄存器）

**6. PC 更新**

```
对齐后的地址 ──→ Multiplexer ──→ new_PC_data ──→ PC 计数器
                     ↑
               is_jalr 信号控制
```

- 当 `is_jalr = 1` 时，选择新地址更新 PC

**7. 控制信号生成**

```
opcode ──→ Comparator(7位) ──┐
                             ├──→ AND Gate ──→ is_jalr
funct3 ──→ Comparator(3位) ──┘
```

- 比较：opcode == 0x67 且 funct3 == 0x0

#### 4.1.2 完整数据流

```
┌─────────────────────────────────────────────────────────────────┐
│                        JALR 数据通路                            │
├─────────────────────────────────────────────────────────────────┤
│                                                                 │
│  指令[31:20] ──→ I_Type_imm ──→ Bit Extender ──→ imm_ext       │
│                                                       │        │
│  指令[19:15] ──→ rs1_addr ──→ 寄存器堆 ──→ rs1_data ──┤        │
│                                                       ▼        │
│                                                  Adder(+)      │
│                                                       │        │
│                                                       ▼        │
│                                             AND Gate(& ~1)     │
│                                                       │        │
│                                                       ▼        │
│                                             new_PC_data ──→ PC │
│                                                                 │
│  PC_data ──→ Adder(+4) ──→ Controlled Buffer ──→ write_data   │
│                    ↑                                            │
│              is_jalr 控制                                       │
└─────────────────────────────────────────────────────────────────┘
```

#### 4.1.3 设计要点总结

1. **双功能**：同时完成跳转和链接（保存返回地址）
2. **地址对齐**：确保 2 字节对齐，符合 RISC-V 规范
3. **灵活寻址**：支持 ±2KB 范围内的间接跳转
4. **典型用法**：
   - `JALR x0, x1, 0`：函数返回（跳转到 ra，不保存返回地址）
   - `JALR x1, x15, 0`：通过函数指针调用（保存返回地址到 ra）

### 4.2 LBU（加载无符号字节）⭐ 重点

**功能描述**：

```
byte_addr   = rs1 + sign_extend(imm[11:0])
byte_offset = byte_addr[1:0] * 8
rd          = zero_extend(mem[byte_addr] >> byte_offset)[7:0]
```

**指令编码**：`I-Type | imm[11:0] | rs1[4:0] | funct3=100 | rd[4:0] | opcode=0000011(0x03)`

#### 4.2.1 设计思路详解

**1. 为什么需要 LBU？**

LBU 用于从内存加载单个字节，主要用于：

- **字符处理**：字符串操作、ASCII 字符读取
- **字节数组访问**：读取字节数组元素
- **I/O 操作**：读取设备寄存器的单字节数据

与 LW 的区别：

- LW 加载 32 位（4 字节）
- LBU 加载 8 位（1 字节），并零扩展到 32 位

**2. 地址计算**

```
  rs1_data ──┐
             ├──→ 32 位 Adder ──→ byte_addr
  imm_ext  ──┘
```

- 与 LW 相同，计算 rs1 + sign_extend(imm)

**3. 字节偏移计算**

```
byte_addr[1:0] ──→ Bit Extender(32→2) ──→ 偏移量(0-3)
                           │
                           ▼
                      Multiplier(*8) ──→ bit_offset(0,8,16,24)
```

- 32 位地址的低 2 位决定字节在字中的位置
- 偏移量 × 8 得到位偏移量

**4. 字节提取**

```
  RAM_data ──┐
             ├──→ Shifter(>>) ──→ 移位后的数据
  bit_offset ┘
```

- 将 32 位 RAM 数据右移 bit_offset 位
- 目标字节位于低 8 位

**5. 零扩展**

```
  移位后的数据 ──┐
                 ├──→ AND Gate ──→ 零扩展后的 32 位数据
  0x000000FF   ┘
```

- 使用 AND 操作与 0xFF 掩码
- 清除高 24 位，保留低 8 位

**6. 写回控制**

```
零扩展后的数据 ──→ Controlled Buffer ──→ write_data ──→ 寄存器堆[rd]
                          ↑
                     is_lbu 信号控制
```

**7. 控制信号生成**

```
opcode ──→ Comparator(7位) ──┐
                             ├──→ AND Gate ──→ is_lbu
funct3 ──→ Comparator(3位) ──┘
```

- 比较：opcode == 0x03 且 funct3 == 0x4

#### 4.2.2 字节选择示例

| 地址低2位 | 偏移量 | 字节位置  | 移位操作    | AND 掩码 |
| --------- | ------ | --------- | ----------- | -------- |
| 00        | 0      | [7:0]     | 不移位      | 0xFF     |
| 01        | 1      | [15:8]    | 右移 8 位   | 0xFF     |
| 10        | 2      | [23:16]   | 右移 16 位  | 0xFF     |
| 11        | 3      | [31:24]   | 右移 24 位  | 0xFF     |

#### 4.2.3 完整数据流

```
┌─────────────────────────────────────────────────────────────────┐
│                        LBU 数据通路                             │
├─────────────────────────────────────────────────────────────────┤
│                                                                 │
│  指令[31:20] ──→ I_Type_imm ──→ Bit Extender ──→ imm_ext       │
│                                                       │        │
│  指令[19:15] ──→ rs1_addr ──→ 寄存器堆 ──→ rs1_data ──┤        │
│                                                       ▼        │
│                                                  Adder(+)      │
│                                                       │        │
│                                                       ▼        │
│                                                   mem_addr     │
│                                                       │        │
│                                                       ▼        │
│                                              RAM 读取数据      │
│                                                       │        │
│                                      ┌────────────────┴───────┐│
│                                      │                        ││
│                                      ▼                        ││
│                             byte_addr[1:0]                    ││
│                                      │                        ││
│                            ┌─────────┴─────────┐              ││
│                            │                   │              ││
│                            ▼                   ▼              ││
│                      Decoder            Multiplier(*8)        ││
│                            │                   │              ││

│                            │                   │              ││
│                            │                   ▼              ││
│                            │            Shifter(>>)           ││
│                            │                   │              ││
│                            │                   ▼              ││
│                            │            AND 0xFF              ││
│                            │                   │              ││

│                            │          write_data ──→ 寄存器堆││
└─────────────────────────────────────────────────────────────────┘
```

#### 4.2.4 设计要点总结

1. **字节选择**：通过地址低 2 位选择目标字节
2. **移位提取**：使用移位器将目标字节移到低 8 位
3. **零扩展**：使用 AND 操作清除高位，实现无符号扩展
4. **典型用法**：
   - `LBU x5, 0(x10)`：从 x10 指向的地址加载一个字节
   - `LBU x6, 3(x10)`：从 x10+3 地址加载一个字节

### 4.3 SB（存储字节）⭐ 重点

**功能描述**：

```
byte_addr   = rs1 + sign_extend(imm[11:0])
byte_offset = byte_addr[1:0]
mem[byte_addr] = rs2[7:0] << (byte_offset * 8)
```

**指令编码**：`S-Type | imm[11:5] | rs2[4:0] | rs1[4:0] | funct3=000 | imm[4:0] | opcode=0100011(0x23)`

#### 4.3.1 设计思路详解

**1. 为什么需要 SB？**

SB 用于向内存存储单个字节，主要用于：

- **字符写入**：字符串操作、字符输出
- **字节数组修改**：修改字节数组元素
- **I/O 操作**：写入设备寄存器的单字节数据

与 SW 的区别：

- SW 存储 32 位（4 字节）
- SB 存储 8 位（1 字节）

**2. S-Type 立即数重组**

```
指令[31:25] ──→ imm[11:5] ──┐
                             ├──→ 12 位立即数
指令[11:7]  ──→ imm[4:0]  ──┘
```

- S-Type 的立即数被分为两部分存储
- 需要在电路中重新组合为完整的 12 位立即数

**3. 地址计算**

```
  rs1_data ──┐
             ├──→ 32 位 Adder ──→ byte_addr
  imm_ext  ──┘
```

**4. 字节偏移计算**

```
byte_addr[1:0] ──→ Bit Extender(32→2) ──→ byte_offset(0-3)
```

**5. 数据移位**

```
  rs2_data[7:0] ──┐
                  ├──→ Shifter(<<) ──→ 移位后的数据
  byte_offset    ┘
                       × 8
```

- 提取 rs2 的低 8 位
- 左移 byte_offset × 8 位

**6. 字节使能生成**

```
byte_offset ──→ Decoder(2位) ──→ BE0, BE1, BE2, BE3
```

- 根据偏移量仅激活一个字节使能

| 偏移量 | 激活的使能 | 写入位置   |
| ------ | ---------- | ---------- |
| 0      | BE0        | [7:0]      |
| 1      | BE1        | [15:8]     |
| 2      | BE2        | [23:16]    |
| 3      | BE3        | [31:24]    |

**7. 内存写入**

```
  移位后的数据 ──→ RAM 数据输入
                        │
  byte_addr    ──→ RAM 地址
                        │
  BE0-BE3      ──→ RAM 字节使能
                        │
  is_sb        ──→ RAM 写使能
```

**8. 控制信号生成**

```
opcode ──→ Comparator(7位) ──┐
                             ├──→ AND Gate ──→ is_sb
funct3 ──→ Comparator(3位) ──┘
```

- 比较：opcode == 0x23 且 funct3 == 0x0

#### 4.3.2 字节写入示例

| 地址低2位 | 偏移量 | 移位量 | 激活的使能 | 写入位置   |
| --------- | ------ | ------ | ---------- | ---------- |
| 00        | 0      | 0      | BE0        | [7:0]      |
| 01        | 1      | 8      | BE1        | [15:8]     |
| 10        | 2      | 16     | BE2        | [23:16]    |
| 11        | 3      | 24     | BE3        | [31:24]    |

#### 4.3.4 设计要点总结

1. **字节定位**：通过地址低 2 位确定目标字节位置
2. **数据移位**：使用移位器将数据移动到正确位置
3. **字节使能**：仅激活目标字节的写使能，保护其他字节
4. **典型用法**：
   - `SB x5, 0(x10)`：将 x5 的低字节存储到 x10 指向的地址
   - `SB x6, 2(x10)`：将 x6 的低字节存储到 x10+2 地址

### 4.4 ADDI（立即数加法）

**功能**：`rd = rs1 + sign_extend(imm)`

**指令编码**：`I-Type | opcode=0x13 | funct3=000`

**实现**：

1. 12 位立即数符号扩展至 32 位
2. 与 rs1 相加
3. 结果写入 rd

**控制信号**：`is_addi = (opcode == 0x13) && (funct3 == 000)`

---

### 4.5 ADD（寄存器加法）

**功能**：`rd = rs1 + rs2`

**指令编码**：`R-Type | opcode=0x33 | funct3=000 | funct7=0x00`

**实现**：

1. 读取 rs1 和 rs2
2. 两个寄存器值相加
3. 结果写入 rd

**控制信号**：`is_add = (opcode == 0x33) && (funct3 == 000) && (funct7 == 0x00)`

---

### 4.6 LUI（加载高位立即数）

**功能**：`rd = imm[31:12] << 12`

**指令编码**：`U-Type | opcode=0x37`

**实现**：

1. 提取高 20 位立即数
2. 低 12 位补零
3. 结果写入 rd

**控制信号**：`is_lui = (opcode == 0x37)`

---

### 4.7 LW（加载字）

**功能**：`rd = mem[rs1 + sign_extend(imm)]`

**指令编码**：`I-Type | opcode=0x03 | funct3=010`

**实现**：

1. 计算地址：rs1 + imm
2. 从 RAM 读取 32 位数据
3. 数据写入 rd

**控制信号**：`is_lw = (opcode == 0x03) && (funct3 == 010)`

---

### 4.8 SW（存储字）

**功能**：`mem[rs1 + sign_extend(imm)] = rs2`

**指令编码**：`S-Type | opcode=0x23 | funct3=010`

**实现**：

1. 计算地址：rs1 + imm
2. 将 rs2 数据写入 RAM
3. 激活所有字节使能

**控制信号**：`is_sw = (opcode == 0x23) && (funct3 == 010)`

---

## 5. 控制信号总结

| 指令 | opcode | funct3 | funct7 | 控制信号 | 写回数据来源  |
| ---- | ------ | ------ | ------ | -------- | ------------- |
| ADDI | 0x13   | 000    | -      | is_addi  | rs1 + imm     |
| JALR | 0x67   | 000    | -      | is_jalr  | PC + 1        |
| ADD  | 0x33   | 000    | 0x00   | is_add   | rs1 + rs2     |
| LUI  | 0x37   | -      | -      | is_lui   | imm << 12     |
| LW   | 0x03   | 010    | -      | is_lw    | 内存数据      |
| LBU  | 0x03   | 100    | -      | is_lbu   | 零扩展字节    |
| SW   | 0x23   | 010    | -      | is_sw    | 无            |
