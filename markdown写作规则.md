# Markdown 写作规则

本文档基于 [markdownlint](https://github.com/DavidAnson/markdownlint) 规则库（v0.40.0）整理，旨在提供一致、规范的 Markdown 写作风格。这些规则有助于提高文档的可读性、可维护性，并确保在不同解析器中的兼容性。

## 核心原则

1. **一致性**：整个文档应使用统一的格式风格。
2. **简洁性**：避免不必要的空格、空行和格式。
3. **结构清晰**：使用正确的标题层级和列表格式。
4. **可访问性**：确保内容对屏幕阅读器等辅助工具友好。

## 常用规则概览

### 标题规则

- **MD001**：标题级别应逐级递增（如 `#` → `##` → `###`），避免跳级。
- **MD003**：在整个文档中使用一致的标题样式（ATX 风格 `#` 或 Setext 风格 `===`/`---`）。
- **MD018**：ATX 风格标题的 `#` 后必须有一个空格（如 `# 标题`）。
- **MD019**：ATX 风格标题的 `#` 后不应有多个空格。
- **MD022**：标题前后应各有一个空行（除非在文档开头或结尾）。
- **MD023**：标题必须从行首开始，不应缩进。
- **MD024**：避免重复标题内容（除非设置 `siblings_only` 参数）。
- **MD025**：整个文档中只能有一个一级标题（`#`）。
- **MD026**：标题末尾不应使用标点符号（句号、逗号、分号、冒号、感叹号等）。
- **MD041**：文件的第一行应是一级标题（可配置允许前置内容）。

### 列表规则

- **MD004**：无序列表应使用统一的符号（`*`、`+` 或 `-`）。
- **MD005**：同一级别的列表项应保持相同的缩进。
- **MD007**：无序列表的嵌套缩进默认为 2 个空格。
- **MD029**：有序列表的前缀应递增（如 `1.`、`2.`、`3.`）或全部使用 `1.`。
- **MD030**：列表标记（`-`、`*`、`+` 或 `1.`）后应有一个空格。
- **MD032**：列表前后应各有一个空行（除非在文档开头或结尾）。

### 代码块规则

- **MD010**：不要使用制表符（Tab）缩进，应使用空格。
- **MD014**：在显示命令的代码块中，如果所有命令都以 `$` 开头，则无需显示 `$` 符号。
- **MD031**：围栏代码块（` ``` `）前后应各有一个空行。
- **MD040**：围栏代码块应指定语言（如 ` ```python `）。
- **MD046**：整个文档中应使用一致的代码块样式（围栏或缩进）。
- **MD048**：整个文档中应使用一致的代码围栏样式（`` ``` `` 或 `~~~`）。

### 空格与空白规则

- **MD009**：避免行尾多余空格（除非用于硬换行，即两个空格）。
- **MD012**：避免连续多个空行（默认最多允许 1 个空行）。
- **MD027**：引用块符号 `>` 后不应有多个空格。
- **MD037**：强调标记（`*` 或 `_`）内不应有空格。
- **MD038**：代码段标记（`` ` ``）内不应有空格。
- **MD039**：链接文本内不应有空格。

### 链接与图片规则

- **MD011**：避免反转的链接语法（如 `(链接)[URL]`）。
- **MD034**：裸 URL 应使用尖括号包裹（如 `<https://example.com>`）。
- **MD039**：链接文本内不应有空格。
- **MD042**：避免空链接。
- **MD045**：图片应提供替代文本（alt text）。
- **MD051**：链接片段（#id）应有效。
- **MD052**：引用链接和图片应使用已定义的标签。
- **MD053**：链接和图片引用定义应被使用。

### 其他规则

- **MD013**：行长度限制（默认 80 字符，可配置）。
- **MD028**：引用块内不应有空行。
- **MD033**：避免使用内联 HTML（可配置允许的元素）。
- **MD035**：整个文档中应使用一致的水平线样式（`---`、`***` 或 `___`）。
- **MD036**：避免使用强调文本代替标题。
- **MD047**：文件应以单个换行符结尾。
- **MD055**：表格管道符样式应一致。
- **MD056**：表格列数应一致。
- **MD058**：表格前后应各有一个空行。

## 配置建议

可以通过项目根目录下的 `.markdownlint.json` 文件自定义规则。例如：

```json
{
  "MD013": false,  // 禁用行长度限制
  "MD007": { "indent": 4 },  // 设置列表缩进为 4 个空格
  "MD024": { "siblings_only": true }  // 允许不同父标题下的重复标题
}
```

## 实践建议

1. **使用编辑器插件**：安装 `markdownlint` 扩展（VS Code 等），实时检查格式问题。
2. **自动修复**：许多规则支持自动修复，可通过编辑器命令执行。
3. **团队协作**：将 `.markdownlint.json` 提交到版本库，确保团队格式一致。
4. **逐步改进**：对于已有文档，可逐步修复问题，避免一次性大规模修改。
